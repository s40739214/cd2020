week10-14

week10

gm用戶登入、API連線設定

1.進入Google API

2.點選建立專案，輸入專案名稱，點選建立

3.點選啟用API和服務，並點選Google+ Domains API

4.點選啟用，點選憑證及設定同意畫面

5.選擇內部，自訂名稱，儲存

6.建立憑證及OAuth用戶端ID

7.選擇網路應用程式

8.將ID及密碼複製到編輯器，命名為oauth_s學號.txt 並存儲在tmp下

9.設定來源及URL，新增URLhttps://localhost:8443、https://localhost:8443/login/google/

10.開啟leo，並打開cd2020pj1.leo

11.git clone cd2020pj1倉儲。

12.將路徑及名稱改為自己設定的

13.完成後存儲，並點選ROC

14輸入pip install authomatic

15進入pj1，打開wsgi.py，即完成。

影片連結：https://youtu.be/6klM1RR-Iso


week12

IPv4 Ubuntu對外連線設定操作

1.點選檔案→喜好設定→網路→新增一個新的NAT網路

2.編輯NAT網路→點選連接埠轉送→新增4個新的連接埠轉送埠號

3.設定

主機IP：127.0.0.1；

客體IP：10.0.2.15；

主機連接埠：22、19999、17443、18443

客體連接埠：22、19999、7443、8443

4.點選設定→網路→設定主機網路為NAT Network

5.設定完後，點選啟動，接著輸入密碼kmol2020

6.開啟LXTermial，輸入ifconfig→輸入sudo apt install net-tools→輸入密碼kmol2020

7.cd tmp->ls->輸入ping 127.0.0.1測試→cd cd2020pj1→git pull

8.輸入sudo vi wsgi.py編輯內容，按i進行編輯

9.將近端的host改為10.0.2.4，接著按Esc並輸入:wq存儲

10.開啟leo，並開啟cd2020pj1.leo，點選Nav搜尋allowExt

11.點選@edituploadform.js，並將第26行新增一個ttt檔，完成後存儲並關閉leo

12.cd..,接著在tmp下新增oauth_scrum.txt，輸入sudo vi oauth_scrum.txt

13.按i編輯，內容先隨便輸入，接著按Esc並輸入:wq存儲

14.cd cd2020pj1，並輸入python3 wsgi.py啟動

15.網址為https:/127.0.0.1:18443/alogin

16.帳號及密碼都為admin，進入後點選fileuploadform，點選要上傳的ttt檔

17.點選download list檢查檔案是否上傳完成

影片連結：https://youtu.be/5Rst5RicUPQ

week14

IPv4遠端控制

下載Xming https://sourceforge.net/projects/xming/

搜尋cmd開啟命令提示字元

輸入ipconfig檢查自己的IPv4

開啟虛擬主機板，開啟檔案裏面的喜好設定，點選網路設定埠號

將前3個埠號設定成電腦IP

完成後按下啟動，密碼為kmol2020

開啟LXTerminal

開啟Xming/XLaunch

開啟putty,設定成和電腦相同的IP

到SSH裡的X11裡，勾選Enable X11 fowarding，並在下方打上localhost:0.0

完成後SAVE並開啟，帳號密碼皆為kmol2020

依序輸入xterm&、cd tmp、cd cd2020pj1、cd docs/cmsimde、vi wsgi.py

按i開始編輯，將近端host改成10.0.2.15，輸入:wq儲存

輸入python3 wsgi.py以開啟近端

網址為https://電腦IP:17443

影片連結：https://youtu.be/PoqXvBK6O0c
